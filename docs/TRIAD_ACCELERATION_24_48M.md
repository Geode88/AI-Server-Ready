# ELARA ZERO-OPS / DFN-0 — Priority Triad Acceleration (24–48 months)

## Purpose
Define a focused acceleration path for the three portfolio clusters with the best ratio of physical pilot probability, reuse potential, and portfolio-wide impact.

## Rule
This triad does not replace or erase the rest of the portfolio.
It functions as a spearhead whose gains should cascade back into the broader synaptic mesh.

## Priority triad
### Triad A — Energy / EMS / control systems
Why this cluster:
- software-first entry point
- credible bridge to physical pilot logic
- strong reuse across infrastructure, runtime, and control patterns

24-month goals:
- formal control architecture
- lightweight dashboard/runtime surface
- first deployment-grade operational view
- readiness pack for a small pilot context

48-month goals:
- pilot-ready control logic package
- stronger digital twin alignment
- clearer physical pilot candidate definition

### Triad B — Built environment / operational module / kitchen or habitat subsystem
Why this cluster:
- segmentable physical target
- strong design-to-spec potential
- realistic path to partial physical realization

24-month goals:
- clear subsystem selection
- spec + BOM + test-plan structure
- readiness assessment for one module
- digital representation and integration plan

48-month goals:
- pilot-ready packet for one subsystem
- stronger installation and operations logic
- better mapping of physical dependencies

### Triad C — Task-specific robotics or DFN-0 preparation layer
Why this cluster:
- high leverage for future execution
- strong linkage to digital lab and pre-industrialization
- useful both as a pilot candidate and as a bridge system

24-month goals:
- define narrow robotic or distributed-preparation target
- structure dependencies and reusable assets
- create readiness and artifact packets
- align with synaptic mesh and digital lab

48-month goals:
- clearer pilot candidate for one narrow target
- stronger execution packet quality
- increased cross-family reuse

## Portfolio protection rule
The rest of the portfolio must remain present through:
- family mapping
- readiness tracking
- gap tracking
- cascade improvement protocol
- reusable asset propagation

## Success condition
The triad should increase:
- physical pilot probability for selected clusters
- standards and artifacts reusable by the rest of the portfolio
- clarity on which additional clusters become next in line
