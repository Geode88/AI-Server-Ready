# ELARA ZERO-OPS / DFN-0 — Cascade Improvement Protocol

## Purpose
Define how improvements made inside the core project should cascade across the broader Elara–Claudio portfolio.

## Principle
Every improvement should be evaluated for reuse beyond its original context.

## Cascade layers
### 1. Structural improvements
Examples:
- registries
- ledgers
- family taxonomy
- synaptic mesh logic
- readiness scoring

### 2. Technical improvements
Examples:
- stack documents
- deploy patterns
- runtime templates
- data model patterns
- prototype structures

### 3. Governance improvements
Examples:
- security policy
- artifact classification
- standard project template
- gap-based prioritization

### 4. Reuse improvements
Examples:
- reusable assets
- common dependencies
- shared patterns across project families

## Rule for applying improvements
For each new improvement ask:
1. Is it reusable?
2. Which family does it strengthen?
3. Which gap does it reduce?
4. Can it become a standard?
5. Can it increase readiness across multiple projects?

## Immediate use
This protocol should be used to:
- propagate stack improvements
- propagate security rules
- propagate synaptic mesh representation
- propagate future artifact templates

## Goal
Turn the portfolio into a progressively more coherent, reusable, and executable system.
