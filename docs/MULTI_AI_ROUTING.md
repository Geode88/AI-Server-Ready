# ELARA ZERO-OPS / DFN-0 — Multi-AI Routing

## Purpose
Define how additional AI systems should be integrated in the future: by role, not by accumulation.

## Routing principles
- keep one primary control layer
- assign specialist models to specific workloads
- avoid model sprawl
- prefer measurable value: cost, speed, context, multimodality, coding, retrieval, or benchmark quality

## Current control layer
- ChatGPT + connected apps + Notion + GitHub

## Candidate future roles
### Control and orchestration
- primary governance and portfolio coordination

### Coding and technical review
- specialist reasoning and coding models

### Multimodal long-context analysis
- models that handle large codebases, documents, audio, video, and PDFs

### Low-cost execution support
- models selected for throughput and budget sensitivity

### Retrieval and ranking support
- embedding, reranking, and search support where needed

### Research benchmark layer
- periodically compare selected external models without changing the main control layer

## Rule
Integrate future models only when they strengthen one of the roles above.
Do not add models just because they exist.
