# ELARA ZERO-OPS / DFN-0 — Stack Map

## Goal
Map the first practical stack for the ELARA ZERO-OPS / DFN-0 program.

## System of record
- Notion: workspace master, registries, ledgers, execution queue
- GitHub: technical source of truth for repo structure, docs, code, and versioning

## Execution and prototype layer
- Replit: rapid software prototyping
- Vercel: deployment and hosting for lightweight web runtime

## Research and low-cost mitigation layer
- Hugging Face: models, datasets, documentation, research support

## Connected context layer
- Gmail: operational communication
- Google Drive: document archive
- Google Calendar: schedule and time layer
- Google Contacts: people and role context

## Initial repository zones
- docs/
- apps/web/
- services/api/
- registry/
- ops/
- artifacts/

## Practical boundaries
- Notion is memory and governance
- GitHub is technical truth
- Replit is prototype speed
- Vercel is deployment surface
- Hugging Face is research support

## Immediate priority stack
1. Notion + GitHub
2. Replit + Vercel
3. Hugging Face as support layer
